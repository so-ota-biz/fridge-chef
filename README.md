## 💡 コンセプト

冷蔵庫にある食材を有効活用し、食品ロスを削減しながら、毎日の献立を考える手間を減らすことを目的としたアプリケーションです。AI が食材の組み合わせやユーザーの好みに合わせて最適なレシピを提案します。

## 📋 要件定義

### プロジェクト目標

- 献立決めのストレス軽減（献立決定時間を 30 分から 10 分へ短縮）
- 食材の有効活用（冷蔵庫の食材を無駄なく使い、食材ロスを削減）
- 料理のマンネリ化解消（多彩なレシピ提案でバリエーションを増やす）
- 料理への自信向上（完成写真の記録で上達を実感し、SNS 投稿の後押し）

### 対象ユーザー

#### プライマリーターゲット

料理に悩む主婦・主夫層

#### セカンダリーターゲット

一人暮らしの社会人、料理初心者

#### 具体的ペルソナ

- 「今日の夕飯何にしよう」と毎日悩む人
- レシピサイトを見ても「この食材がない」となる人
- 買い物に行く前に、家にあるもので済ませたい人
- 作った料理を SNS に投稿したい人

### 解決したい課題

1. 献立決めの時間とストレス（毎日の負担を軽減）
2. 食材の無駄（冷蔵庫の食材を使い切れず捨ててしまう）
3. 料理のマンネリ化（同じメニューの繰り返し）
4. 料理への自信不足（上達を実感しづらい）
5. SNS 映えする工夫（見栄えの良い写真作り）

### システムの特徴・差別化ポイント

- 冷蔵庫ベースの料理提案（保有食材から最適なレシピを AI が自動生成）
- カテゴリ別アイコン選択（直感的な操作で簡単に食材を選択可能）
- 3 パターンのレシピ提案（条件に合わせた複数の選択肢を提示）
- 調理記録の保存（写真付きで記録し、振り返りが可能）
- AI 生成の完成イメージ（レシピごとに完成写真を自動生成）

## ✨ 主な機能

### 1. 認証機能

- ユーザー登録・ログイン
- ローカルストレージベースJWT認証によるセキュアなセッション管理

### 2. 食材選択

- カテゴリ別の食材マスタから選択
- カスタム食材の手動入力

### 3. レシピ生成条件の設定

- ジャンル（和食・洋食・中華など）
- 難易度（簡単・普通・むずかしい）
- 調理時間（15 分・30 分・60 分 など）
- 人数（1〜10 人）

### 4. AI レシピ提案

- 選択した食材と条件に基づき、AI が 3 つのレシピを生成
- レシピ詳細（材料・手順・調理時間など）を表示
- レシピ画像を自動生成

### 5. 調理記録管理

- 作ったレシピの記録作成
- 評価（星 5 段階）とメモの保存
- 調理記録の一覧表示・詳細確認・編集・削除

### 6. レスポンシブデザイン

- スマートフォン、タブレット、デスクトップに対応
- Mantine UI によるシンプルな UI/UX

## 🛠 使用技術

### Frontend

- フレームワーク: Next.js (App Router)
- 言語: TypeScript
- UI ライブラリ: Mantine UI
- 状態管理: Zustand
- データフェッチング: TanStack Query (React Query)
- HTTP クライアント: Axios
- フォーム管理: React Hook Form + Zod

### Backend

- フレームワーク: NestJS
- 言語: TypeScript
- ORM: Prisma
- データベース: Supabase (PostgreSQL)
- 認証: Passport + JWT
- パスワードハッシュ: bcrypt
- バリデーション: class-validator + class-transformer
- 画像処理: Sharp
